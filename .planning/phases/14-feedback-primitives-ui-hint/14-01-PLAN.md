<objective>
Make every user-initiated action in the CRATE UI emit consistent feedback through the single existing `#toast` primitive, and guarantee the add-item modal always opens with Type = "album".

Purpose: Today, per-item "Scan Now" silently updates only the floating scan panel and never emits a toast, so on a successful scan the user sees a pill collapse with no clear "X new listings found" confirmation — inconsistent with the "Item added, scanning…" flow (which does toast) and with the "Scan All" flow (which toasts via redirect). We also want the modal Type to always default to "album" on open, since the dashboard use case is overwhelmingly album lookups.

Output: Updated `templates/base.html` (adds `window.showToast()` helper + wires scan completion to it) and `templates/index.html` (resets modal type on open). Zero backend changes; iOS Shortcut API contract untouched.
</objective>
