### Spacing tokens — OVERRIDE for Phase 4 (compressed)

These tokens override Phase 3 spacing for card-specific contexts:

```css
/* Phase 4 overrides — applied via component classes, not :root tokens */
/* --space-md remains 16px globally; card padding uses 12px via .card-body */
/* --space-lg remains 24px globally; card grid uses 12px via .card-grid gap */
/* --space-xl remains 32px globally; section gap uses 20px via .section-gap */
```

Concrete values:
- `.card-body` padding: `12px` (was `var(--space-md)` = 16px). Source: 04-CONTEXT.md D-07.
- `.card-grid` gap: `12px` (was `var(--space-lg)` = 24px). Source: 04-CONTEXT.md D-07.
- `.section-gap` gap (between major page sections): `20px` (was 24px). Source: 04-CONTEXT.md D-07.

The global `--space-*` tokens retain their Phase 3 values. Only these three component-level overrides compress spacing.

## Typography

Unchanged from Phase 3 for body text. One addition: wordmark typography.

| Token | Role | Size | Weight | Line Height | Usage |
|-------|------|------|--------|-------------|-------|
| `--text-sm` | Label/Meta | 14px | 400 | 1.5 | Form labels, table headers, badge text, card metadata, timestamps |
| `--text-body` | Body | 16px | 400 | 1.5 | Default paragraph, listing rows, modal body |
| `--text-heading` | Heading | 24px | 600 | 1.2 | Item detail page H1, section headings |
| `--text-price` | Price display | 20px | 600 | 1.2 | AUD total in detail page listing cards |

**Weights used:** 400 (regular) and 600 (semibold) only. No other weights.

### CRATE Wordmark (D-08)

```css
.nav-brand {
  font-size: 16px;        /* uses --text-body size; letter-spacing 0.25em and weight 600
                             provide sufficient wordmark differentiation without a distinct size */
  font-weight: 600;        /* semibold — same weight as headings; letter-spacing provides
                             wordmark differentiation */
  letter-spacing: 0.25em; /* wide track — spells out as "C R A T E" visually */
  text-transform: uppercase;
  line-height: 1;          /* single-line wordmark, no vertical bleed */
  color: var(--color-accent); /* white — the primary accent application */
}
```

Template text: `CRATE` (single word, uppercase forced in CSS via `text-transform`, not encoded as `C R A T E` in HTML — the letter-spacing does the visual work).

Source: 04-CONTEXT.md D-08; font-size revised to `--text-body` (16px) to keep type scale at 4 sizes; weight revised to 600 to keep weight count at 2 (checker fix).

Font declaration (unchanged from Phase 3):
```css
--font-sans: -apple-system, BlinkMacSystemFont, "Segoe UI", Helvetica, Arial, sans-serif;
```

## Color — 60/30/10 Split

| Role | Hex / Token | Coverage | Usage |
|------|-------------|----------|-------|
| Dominant (60%) | `#0a0a0a` (`--color-bg`) | Page background, nav bar, table header rows, input backgrounds, modal backdrops | Full-bleed page surface — near-black neutral, no blue cast |
| Secondary (30%) | `#111111` (`--color-surface`) | Cards (dashboard grid), panels (scan progress, stats bar), modal panels, listing table container | All contained surfaces |
| Accent (10%) | `#ffffff` (`--color-accent`) | See reserved-for list below | Intentional attention elements only — strict list |
| Destructive | `#f87171` (`--color-destructive`) | Remove from Crate button text and border; confirm copy | Destructive actions exclusively |

**Accent reserved for (explicit list) — white `#ffffff`:**
1. Nav bar wordmark text (`CRATE`)
2. Primary CTA button background (Add Item modal submit, Scan All, Scan Now, Save Changes)
3. Best price value in card and detail table (`.price-best` class)
4. "View Deals →" link on dashboard cards
5. "← Back to Dashboard" link on detail page
6. Spinner border top segment (`.spinner`)
7. Pulse border animation on scanning cards (`.card--scanning`)
8. Form input focus ring
9. Notify-below-pct label highlight in detail header

Accent is NOT used for: general text, all borders, badge colours, table row hover, secondary buttons, source badges.

Source: 04-CONTEXT.md D-03, D-04.

### Palette Aesthetic Target

Linear / Vercel / Apple dark mode territory: monochrome, high contrast, cold. Not warm, not vintage. `#0a0a0a` reads as "true black" on most displays. The white accent creates sharp figure-ground separation with no colour noise.

Source: 04-CONTEXT.md Specifics section.

## Layout Contracts

### Dashboard Grid

Unchanged from Phase 3, except grid gap compressed to 12px:

```
Mobile  (<768px):  1 column
Tablet  (≥768px):  2 columns
Desktop (≥1280px): 4 columns
```

- Grid gap: `12px` (was `--space-lg` = 24px). Source: D-07.
- Card body padding: `12px` (was 16px). Source: D-07.
- Card aspect: artwork fills card face at 1:1 aspect ratio via `aspect-ratio: 1 / 1` + `object-fit: cover`.
- Below artwork: item title (`--text-sm`, 14px, weight 600) and best AUD price (`--text-sm`, 14px, weight 400, white accent).
- Card click target: entire card navigates to `item_detail.html`. Source: Phase 3 D-05 (preserved).
- Max content width: 1280px, centred, `padding-inline: --space-md`.

### Card Hover/Focus States

After palette swap, accent is white — adjust hover shadow accordingly:

- Hover: `transform: translateY(-2px)` + `box-shadow: 0 8px 32px rgba(0,0,0,0.6)` — 200ms ease. (Deeper shadow against near-black bg.)
- Focus (keyboard): `outline: 2px solid var(--color-accent)` (white) offset 2px.
- Scanning card: `pulse-border` keyframe animation with white accent — update keyframe stop colours.

Updated `pulse-border` keyframe:
```css
@keyframes pulse-border {
  0%, 100% { border-color: rgba(255,255,255,0.2); }
  50%       { border-color: rgba(255,255,255,0.6); }
}
```

Source: Claude's discretion — hover/focus adjustments after palette swap.

### Item Detail Page

Unchanged from Phase 3 structurally. Visual changes from token updates only:
- Header artwork thumbnail: `120px × 120px`, `border-radius: 0` (sharp).
- Listing table container: `border-radius: 0` (sharp).
- AUD total: white accent (`.price-best` — now white, not amber).

### Artwork Quality Upgrade (D-09)

Backend change only — no template or CSS contract change. The `artwork_url` stored on `WishlistItem` will now point to `images[0].uri` from the Discogs release endpoint (higher resolution) rather than the search `cover_image` thumbnail. The template `<img src="/api/artwork?url={{ item.artwork_url }}">` pattern is unchanged.

- Executor must call Discogs release endpoint (`/releases/{id}`) after a match is found, extract `images[0].uri`.
- The artwork proxy endpoint (`GET /api/artwork`) is unchanged.
- `onerror` fallback to `/static/vinyl-placeholder.svg` is unchanged.

## Interaction Contracts

Unchanged from Phase 3. Carried forward:

### Add Item Modal
- Triggered by: "Add Item" button in nav bar.
- Fields: query (text, required), type (select: album/artist/label/subject), notes (text, optional), notify_below_pct (number, 1–90, default 20), notify_email (checkbox).
- Submit: `POST /wishlist/add`.
- Cancel: close modal. Backdrop click: close modal.

### Edit Item
- Accessible from detail page only (not from card face). Source: Phase 3 D-05 (preserved).
- Submit: `POST /wishlist/{id}/edit`.

### Delete Item
- Triggered by: "Remove from Crate" button on item detail page.
- Method: `POST /wishlist/{id}/delete`.
- Confirmation: `confirm("Remove this item from your crate?")` — browser dialog, no separate modal.

### Scan Trigger
- Dashboard: "Scan All" — shows floating scan pill (preserved from Phase 1).
- Detail: "Scan Now" — same.
