> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> Stack: Python/FastAPI, Jinja2 templates, vanilla CSS with custom properties. No component library.

## Spacing Scale

Declared values (all multiples of 4). No changes from Phase 10 tokens — these are pre-existing and correct.

| Token | Value | Usage |
|-------|-------|-------|
| `--space-xs` | 4px | Icon gaps, badge padding, inline separators |
| `--space-sm` | 8px | Button padding block, compact element spacing |
| `--space-md` | 16px | Default element spacing, card padding, container inline padding |
| `--space-lg` | 24px | Section padding, modal padding |
| `--space-xl` | 32px | Layout gaps, grid row gaps |
| `--space-2xl` | 48px | Major section breaks |
| `--space-3xl` | 64px | Page-level vertical rhythm |

Exceptions:
- Touch targets (buttons, card links): minimum 44px height — already enforced on `.btn-cta`. Extend to all interactive controls.
- Container `padding-inline`: reduce from `var(--space-md)` to `var(--space-sm)` on the card grid wrapper only, to push cards closer to viewport edge (D-08).

## Typography

Phase 11 reverts the Phase 10 scale expansion (D-09). Artwork is the visual hero — text is subordinate.

| Role | Token | Size | Weight | Line Height | Usage |
|------|-------|------|--------|-------------|-------|
| Label | `--text-label` | 12px | 400 | 1.4 | Badges, metadata, timestamps, source labels |
| Body | `--text-body` | 16px | 400 | 1.5 | Card titles, table cells, form inputs, paragraphs |
| Subheading | `--text-subheading` | 20px | 600 | 1.2 | Section headings (H2), stats bar values |
| Heading | `--text-heading` | 24px | 700 | 1.1 | Page-level H1, modal titles |
| Display | n/a | chosen font, ~24–28px | 700 | 1.0 | `.nav-brand` CRATE wordmark only |

Weights used: 400 (regular) and 600/700 (semibold/bold). No other weights.

Notes:
- The Phase 10 `--text-title` (21px) and `--text-price` (28px) tokens are preserved for backward compatibility but deprioritised in templates. Card album names drop back to `--text-body` (16px).
- Price display remains at `--text-price` (28px) — it is a functional data signal, not decoration.
- Source: D-09 (revert to pre-Phase-10 scale).

## Color

### Palette Philosophy

B&W at rest. Colour only on hover/interaction. Album artwork is the only persistent source of colour on the page. This creates a "gallery wall" feel — monochrome canvas that activates on engagement.

Source: D-03, D-04, D-05.

### Token Changes for This Phase

| Token | Current Value | New Value | Reason |
|-------|--------------|-----------|--------|
| `--color-bg` | `#0a0a0a` | `#0d0b0a` | Warm dark shift (D-05) |
| `--color-surface` | `#111111` | `#141210` | Warmer card surface, slight lift from bg |
| `--color-surface-alt` | `#0a0a0a` | `#0d0b0a` | Match new bg |
| `--color-success` | `#34d399` (vivid) | `#5a9e7a` (muted green) | Deal colour: visible on hover only, never vibrant at rest (D-04) |
| `--color-destructive` | `#f87171` (vivid) | `#c46060` (muted red) | Destructive colour: visible on hover only (D-04) |

All other tokens carry forward unchanged.

### 60 / 30 / 10 Split

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0d0b0a` | Page background, nav background, modal overlay backdrop |
| Secondary (30%) | `#141210` | Cards, sidebar (none currently), typeahead dropdown, modal surface |
| Accent (10%) | `#ffffff` | See reserved list below |
| Muted green | `#5a9e7a` | Deal hover state only (badge reveal, card hover border) |
| Muted red | `#c46060` | Destructive hover state only (delete button hover, confirm hover) |

### Accent Reserved For

White (`#ffffff` / `--color-accent`) is used ONLY on:
1. `.nav-brand` CRATE wordmark
2. Primary CTA button background (`.btn-cta` — white bg, dark text)
3. Focus-visible rings (`box-shadow: 0 0 0 2px var(--color-accent)`)
4. Active/selected state borders
5. Logo icon (if monochrome SVG)

Accent is NOT used as a general highlight colour on cards, badges, or body text.

### Deal Badge — B&W at Rest, Colour on Hover

- At rest: `.card--deal` badge is hidden (`opacity: 0`, `visibility: hidden`)
- On `.card:hover`: badge fades in (`opacity: 1`, `visibility: visible`, `transition: opacity 150ms`)
- Badge colour on reveal: `--color-success` (muted green `#5a9e7a`), not vivid
- No coloured left border on deal cards at rest — removed in this phase (D-06)

### Ghost Cards (Empty / No Listings)

- `.card--empty`: `opacity: 0.35` (within D-08's 30–40% range)
- No border
- Artwork faded to match opacity
- Placeholder image: new asset file provided by user — token `--img-placeholder` in CSS pointing to the new file path

Source: D-07.

## Interaction Contract

### Hover Effects (D-12, D-13, D-14)

Energy level: subtle and refined base, with occasional brutalist sharpness. No bounce, no elastic easing. No decorative animation — every motion must serve a purpose.

| Element | Rest State | Hover State | Transition |
|---------|-----------|-------------|------------|
| Card (`.wishlist-card`) | No transform, no shadow | `translateY(-2px)`, subtle shadow `0 4px 16px rgba(0,0,0,0.4)` | `transform 150ms ease-out, box-shadow 150ms ease-out` |
| Deal badge | `opacity: 0; visibility: hidden` | `opacity: 1; visibility: visible` | `opacity 150ms ease-out` |
| CTA button | white bg | `--color-accent-hover` (#e0e0e0) bg | `background 150ms` (existing) |
| Secondary button | transparent, white border | white bg, dark text | `background 150ms, color 150ms` |
| Destructive button | transparent, muted-red border | muted-red bg, white text | `background 150ms, color 150ms` |
| Delete confirm button | hidden | fade in inline below card title | `opacity 120ms` |
| Nav brand | static | no hover effect — wordmark only | none |

### Micro-interactions

| Element | Trigger | Effect |
|---------|---------|--------|
| Any `.btn-*` | `:active` press | `transform: scale(0.97)` — instant cut in, ease-out on release |
| Card link | `:active` | `opacity: 0.8` flash |
| Add item form submit | success | brief opacity fade on the newly added card appearing |
| Scan button | disabled state | `opacity: 0.5; cursor: not-allowed` — no hover lift |

### Page Transitions

No full-page transition framework. Navigation is server-rendered — use `<meta name="view-transition">` if browser support is acceptable, otherwise skip. Prefer per-element reveals over page-level animation.

### Scrollbar Styling (D-17)

Applied via `::-webkit-scrollbar` on the typeahead dropdown and `body` / `html`. B&W palette only.

| Property | Value |
|----------|-------|
| Width (vertical) | 6px |
| Track colour | `#1a1a1a` (near bg) |
| Thumb colour | `#444444` at rest |
| Thumb colour on hover | `#666666` |
| Border radius | 3px (minimal rounding — consistent with brutalist aesthetic) |

### Image Loading State (D-16)

- Card artwork: show a skeleton pulse (CSS animation, `background: linear-gradient(90deg, #1a1a1a, #2a2a2a, #1a1a1a)`, `background-size: 200%`, `animation: skeleton-pulse 1.2s ease-in-out infinite`) while `<img>` is loading
- After `<img onload>` fires: fade in image, remove skeleton (`opacity: 0 → 1, transition: opacity 200ms`)
- After adding a new item via form: JS refreshes the card's `<img>` src without full page reload — appends `?t={timestamp}` cache-bust parameter

### Default Form Values (D-18)

- Add-item modal: `item_type` select defaults to `album` — set via `selected` attribute in Jinja2 template
- No other form default changes in this phase

## Logo Contract (D-01, D-02)

- Concept: abstract vinyl-on-shelf — diagonal lines evoking records standing side-on in a crate
- Format: SVG (monochrome, single-colour — white `#ffffff` on dark bg, or inverted)
- Placement: left of `.nav-brand` wordmark in `.nav-inner`
- Size: 28×28px at 1x — scales with nav height (64px nav, logo vertically centred)
- Generation: AI tooling produces multiple options during planning; user selects before implementation
- Email: same logo included in `deal_alert.html` (D-11) — use inline SVG or hosted static URL with fallback
