# Plan 18-02: Item detail placeholder swap (FIX-02)

## Objective
Replace both `vinyl-placeholder.svg` references in `templates/item_detail.html` with the new `empty-vinyl-placeholder.png` asset. Two sites: the `{% else %}` branch (no artwork) and the `onerror` attribute fallback.

## Must-Haves
- No remaining references to `vinyl-placeholder.svg` in `templates/item_detail.html`
- Both placeholder sites point to `/static/empty-vinyl-placeholder.png`
- 120x120 render size, `object-fit: cover`, `border-radius: 0`, and `alt` text preserved

## Tasks

<task id="T1">
<title>Swap placeholder to empty-vinyl-placeholder.png in both branches</title>
<read_first>
- templates/item_detail.html
</read_first>
<action>
In `templates/item_detail.html`, make two exact string replacements inside the artwork block (lines 14–29):

1. On line 19, replace:
```
onerror="this.src='/static/vinyl-placeholder.svg'"
```
with:
```
onerror="this.src='/static/empty-vinyl-placeholder.png'"
```

2. On line 24, replace:
```
            src="/static/vinyl-placeholder.svg"
```
with:
```
            src="/static/empty-vinyl-placeholder.png"
```

Do NOT change:
- The surrounding `{% if item.artwork_url %}` / `{% else %}` structure
- `alt` text (`"{{ item.query }} album cover"` and `"{{ item.query }} — no artwork available"` stay as-is)
- The inline `style="width: 120px; height: 120px; object-fit: cover; border-radius: 0;"` attributes on both `<img>` tags
- Any other part of the file (header card, best deals section, listings table, edit modal, scripts — all out of scope)

Do not add `loading="lazy"` or any other attributes — UI-SPEC lists it as a property but the existing images have no loading attribute and the spec's "Implementation Checklist" does not require adding it. Preserve behaviour parity; only swap the two paths.
</action>
