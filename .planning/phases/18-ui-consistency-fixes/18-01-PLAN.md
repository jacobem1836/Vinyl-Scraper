# Plan 18-01: Toast-only post-add confirmation (FIX-01)

## Objective
Eliminate the secondary scan panel UI that appears alongside the toast after a user adds an item. Only the `#toast` primitive should confirm the add. The scan panel must only appear when a scan is actively running.

## Must-Haves
- Only `#toast` is visible after item add — the `#scanPanel` element stays hidden unless a scan is actively running
- Toast copy uses em-dash: "Item added — scanning in background"
- Existing scan-panel behaviour while scans ARE running is preserved (no regression to live scan UI)

## Tasks

<task id="T1">
<title>Gate scan panel auto-activation to is_running only</title>
<read_first>
- templates/base.html
</read_first>
<action>
In `templates/base.html`, locate the DOMContentLoaded handler inside the scan-panel IIFE (around line 255–260). The current code is:

```js
document.addEventListener('DOMContentLoaded', function() {
  fetch('/api/scan/status')
    .then(function(r) { return r.json(); })
    .then(function(s) { if (s.is_running) { showPanel(); startPolling(); } else if (s.started_at) { showPanel(); renderStatus(s); } })
    .catch(function(){});
});
```

Replace the `.then(function(s) { ... })` handler so that `showPanel()` is ONLY called when `s.is_running === true`. Remove the `else if (s.started_at)` branch entirely — do not call `showPanel()` or `renderStatus(s)` when a scan is merely completed-but-not-dismissed. The replacement must read exactly:

```js
.then(function(s) { if (s.is_running) { showPanel(); startPolling(); } })
```

Do not modify any other code in this file. Do not touch the `renderStatus()` function, `showPanel()`, `hidePanel()`, or the `?toast=` query-param handler.
</action>
<acceptance_criteria>
- `grep -n "else if (s.started_at)" templates/base.html` returns no matches
- `grep -nE "if \(s\.is_running\) \{ showPanel\(\); startPolling\(\); \} \}\)" templates/base.html` returns exactly one match inside the DOMContentLoaded block (around line 258)
- `grep -c "showPanel" templates/base.html` returns the same count MINUS 1 compared to the pre-change file (one call site removed)
- Existing `renderStatus` function body remains unchanged — `grep -n "function renderStatus" templates/base.html` still present
- The `?toast=` handler block (lines ~108–122) is unchanged — `grep -n "params.get('toast')" templates/base.html` still returns a match
</acceptance_criteria>
</task>

<task id="T2">
<title>Update add-item redirect toast copy to use em-dash</title>
<read_first>

<action>
In `app/routers/wishlist.py` at line 121, the current redirect is:

```python
return RedirectResponse(url="/?toast=Item+added%2C+scanning+in+background", status_code=303)
```

Replace the URL with the em-dash version. The em-dash character is `—` (U+2014), URL-encoded as `%E2%80%94`. The new line must be exactly:

```python
return RedirectResponse(url="/?toast=Item+added+%E2%80%94+scanning+in+background", status_code=303)
```

This replaces the comma+space (`%2C+`) with space+em-dash+space (`+%E2%80%94+`). Do not modify any other redirect in this file (lines 148, 174, 202 stay as-is — those are out of scope per UI-SPEC).
</action>
