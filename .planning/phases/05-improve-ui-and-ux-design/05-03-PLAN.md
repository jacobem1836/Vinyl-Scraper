**Part C — Focus management in base.html modal JS:**

In the `openModal` function in base.html, after the `requestAnimationFrame` block that removes opacity-0 and translate-y-4 classes, add focus to the first focusable element. The updated `openModal` function should be:

```javascript
const openModal = () => {
  if (!modal) return;
  modal.classList.remove('hidden');
  requestAnimationFrame(() => {
    modal.dataset.open = 'true';
    const backdrop = modal.querySelector('[data-backdrop]');
    const panel = modal.querySelector('[data-panel]');
    if (backdrop) backdrop.classList.remove('opacity-0');
    if (panel) panel.classList.remove('opacity-0', 'translate-y-4');
    const firstFocusable = modal.querySelector('input, select, textarea, button');
    if (firstFocusable) firstFocusable.focus();
  });
};
```

In the `closeModal` function, after the setTimeout that adds the 'hidden' class, return focus to the trigger button. The updated `closeModal` function should be:

```javascript
const closeModal = () => {
  if (!modal) return;
  modal.dataset.open = 'false';
  const backdrop = modal.querySelector('[data-backdrop]');
  const panel = modal.querySelector('[data-panel]');
  if (backdrop) backdrop.classList.add('opacity-0');
  if (panel) panel.classList.add('opacity-0', 'translate-y-4');
  setTimeout(() => modal.classList.add('hidden'), 200);
  if (openBtn) openBtn.focus();
};
```

  <action>
Replace the delete form on the detail page (~line 51-53) that currently uses `onsubmit="return confirm('Remove this item from your crate?')"`.

Current HTML:
```html
<form action="/wishlist/{{ item.id }}/delete" method="post" onsubmit="return confirm('Remove this item from your crate?')">
  <button type="submit" class="btn-destructive">Remove from Crate</button>
</form>
```

Replace with an inline confirmation pattern using two states:

```html
<div id="deleteConfirm" style="display: flex; align-items: center; gap: var(--space-sm);">
  <button type="button" id="deleteInitBtn" class="btn-destructive" onclick="document.getElementById('deleteInitBtn').style.display='none'; document.getElementById('deleteConfirmBtns').style.display='flex';">Remove from Crate</button>
  <div id="deleteConfirmBtns" style="display: none; align-items: center; gap: var(--space-sm);">
    <form action="/wishlist/{{ item.id }}/delete" method="post" style="display: inline;">
      <button type="submit" class="btn-destructive">Confirm?</button>
    </form>
    <button type="button" class="btn-secondary" onclick="document.getElementById('deleteInitBtn').style.display='inline-flex'; document.getElementById('deleteConfirmBtns').style.display='none';">Cancel</button>
  </div>
</div>
```

This pattern:
1. Shows "Remove from Crate" button initially
2. On click, hides it and shows "Confirm?" + "Cancel" buttons inline
3. "Confirm?" submits the delete form (no JS confirm dialog)
4. "Cancel" resets back to the initial state
5. No timeout — stays in confirm state until user acts (per Claude's discretion from CONTEXT.md)
