## Design System

| Property | Value |

## Spacing Scale

All tokens are defined in `static/style.css` under `:root`. This phase replaces hard-coded `12px` values with these tokens (Priority 7).

| Token | Value | Usage |
|-------|-------|-------|
| `--space-xs` | 4px | Icon gaps, inline padding, badge padding |
| `--space-sm` | 8px | Compact element spacing, card-body gap, button padding (vertical) |
| `--space-md` | 16px | Default element spacing, button padding (horizontal), card-body padding (post-fix) |
| `--space-lg` | 24px | Section padding, modal padding |
| `--space-xl` | 32px | Layout gaps |
| `--space-2xl` | 48px | Major section breaks, empty-state padding (vertical) |
| `--space-3xl` | 64px | Page-level spacing |

**Exceptions:**
- Touch targets: `min-height: 44px` on all three button classes (Priority 4). This is an absolute minimum, not a spacing token.
- Nav height: 64px (existing, unchanged).
- Form input padding: `var(--space-sm) 12px` — the horizontal `12px` is intentionally kept in `.form-input` as it predates this phase's scope. Only `.card-body` padding and `.card-grid gap` are targeted for the 12px replacement.

**Priority 7 decisions (Claude's discretion):**
- `.card-body { padding: 12px }` → replace with `var(--space-md)` (16px). The card body needs comfortable reading space; 8px would be too tight.
- `.card-grid { gap: 12px }` → replace with `var(--space-sm)` (8px). Grid gutters should be tight in a brutalist layout; 16px would over-expand the grid visually.

## Typography

Source: `static/style.css` `:root` tokens. Priority 5 changes `--text-heading` and introduces a new H2 size distinction.

| Role | Token / Size | Weight | Line Height | Usage |
|------|-------------|--------|-------------|-------|
| Small / Label | `--text-sm` = 14px | 400 (regular) | 1.5 | Secondary info, badges, timestamps, price sub-labels, table cells |
| Body | `--text-body` = 16px | 400 (regular) | 1.5 | Body copy, form labels, card titles |
| Price | `--text-price` = 20px | 600 (semibold) | 1.2 | Landed price display in Best Deals cards |
| Heading H2 / Section title | 22px (inline `font-size`) | 600 (semibold) | 1.2 | "Best Deals", "All Listings" section headings on detail page; modal titles ("Add Wishlist Item", "Edit Wishlist Item") |
| Heading H1 / Page title | `--text-heading` = **28px** (was 24px) | 600 (semibold) | 1.2 | Item name `<h1>` on detail page; empty-state heading "Your crate is empty" |

**Priority 5 change (locked):**
- `--text-heading` bumped from 24px to 28px
- H2 section headings set to 22px (Claude's discretion: 22px chosen over 24px to preserve a clear 6px step below the 28px H1, while remaining above body text)
- Creates a genuine 28px (H1) → 22px (H2) → 16px (body) hierarchy

**Weight constraint:** Exactly 2 weights used — 400 (regular) for body copy, 600 (semibold) for headings, prices, button labels. No 500 weight introductions in this phase.

### 60/30/10 Split

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0a0a0a` | Page background (`--color-bg`), surface-alt, nav background |
| Secondary (30%) | `#111111` | Cards, modal panels, table containers, scan card, toast (`--color-surface`) |

**Accent reserved for (explicit list):**
1. Nav brand "CRATE" logotype
2. `.btn-cta` background fill (the only filled/inverted button)
3. `:focus-visible` ring on all three button classes (2px `box-shadow` in `--color-accent`)
4. Best price display (`.price-best` class, `.text-accent`)
5. "View Listing →" and "Open" links on detail page
6. "← Back to Dashboard" link on detail page
7. Notify threshold percentage highlight in item meta
8. Inline "Add Item" text emphasis in empty state
9. Form input `:focus` ring (already existing, unchanged)
10. Scan spinner top border (`border-top-color: var(--color-accent)`)

Accent is NOT used for: general hover backgrounds, secondary button borders, muted text, timestamps, or decorative elements.

### Priority 1 — Contrast fix (locked)

`--color-text-faint` changes from `#555555` to `#686868`. This achieves ~4.5:1 contrast ratio on `#0a0a0a` background, meeting WCAG AA at 14px. Affects: "No listings found yet", price sub-labels (base price + shipping line), timestamps, separator pipes ("|") in stats bar.

### Priority 9 — Card hover shadow (locked)

Current: `box-shadow: 0 8px 32px rgba(0, 0, 0, 0.6)` — invisible on `#0a0a0a`.

Replacement:
```css
.card:hover {
  box-shadow: 0 0 0 1px rgba(255,255,255,0.1), 0 8px 24px rgba(0, 5, 20, 0.8);
}
```

The `0 0 0 1px rgba(255,255,255,0.1)` layer creates a subtle white aura ring; the second layer provides depth. Both read on near-black backgrounds.

## Interaction Contract

### Button States — Full State Matrix

All three button classes (`.btn-cta`, `.btn-secondary`, `.btn-destructive`) must implement all four states:

| State | Current | Phase 5 Addition |
|-------|---------|-----------------|
| Default | defined | unchanged |
| `:hover` | defined | unchanged |
| `:focus-visible` | MISSING | add `box-shadow: 0 0 0 2px var(--color-accent); outline: none;` |
| `:active` | MISSING | add `transform: scale(0.97);` |

**Priority 2 — Focus-visible (locked):**
```css
.btn-cta:focus-visible,
.btn-secondary:focus-visible,
.btn-destructive:focus-visible {
  outline: none;
  box-shadow: 0 0 0 2px var(--color-accent);
}
```
Matches the existing `form-input:focus` pattern (already uses this exact approach).

**Priority 8 — Active state (locked):**
```css
.btn-cta:active,
.btn-secondary:active,
.btn-destructive:active {
  transform: scale(0.97);
}
```
No transition duration needed — `:active` is instantaneous by convention. The 200ms transition on the button already applies to `background` only; `transform` inherits the base transition if it is broadened, otherwise add `transition: background 200ms, transform 100ms`.

**Priority 4 — Touch targets (locked):**
- `.btn-cta` currently has `min-height: 40px` — bump to `44px`
- `.btn-secondary` has no `min-height` — add `min-height: 44px`
- `.btn-destructive` has no `min-height` — add `min-height: 44px`

**Claude's discretion — `.btn-cta:disabled`:** Include the disabled state fix only if it is a single CSS rule (`opacity: 0.5; cursor: not-allowed; pointer-events: none`). If it requires template changes, skip for this phase.

### Modal Accessibility

**Priority 3 — Focus management (locked):**

In `base.html` `openModal()` function, after the `requestAnimationFrame` animation call, add:
```js
modal.querySelector('input, select, textarea, button').focus();
```

In `base.html` `closeModal()` function, after the timeout, add:
```js
openBtn.focus();
```

Same pattern in `index.html` `openEdit()` / `closeEdit()` functions.

**ARIA attributes (locked):**
- Add `role="dialog"` to `.modal-panel` (the `data-panel` / `data-edit-panel` divs)
- Add `aria-modal="true"` to `.modal-panel`
- Add `aria-labelledby="addItemModalTitle"` (or `editItemModalTitle`) to `.modal-panel`
- Add matching `id` attributes to the modal `<h2>` elements

### Delete Confirmation — Inline Pattern

**Priority 10 — Inline confirmation (locked):**

Replace `onsubmit="return confirm('Remove this item from your crate?')"` with an inline two-step pattern.

**Behaviour contract:**
1. Initial state: single "Remove from Crate" button (`.btn-destructive`)
2. First click: button group appears — "Confirm?" (`.btn-destructive`) + "Cancel" (`.btn-secondary`) side by side
3. "Confirm?" click: form submits
4. "Cancel" click: resets to initial state
5. No timeout — the state persists until the user explicitly cancels or confirms
6. No browser `confirm()` dialog — entirely inline within the page

**Implementation approach (Claude's discretion):**
Toggle a CSS class or `data-confirming` attribute on the form wrapper. Show/hide via the `.hidden` utility class already in the design system. No new CSS classes needed — use existing `.btn-destructive`, `.btn-secondary`, `.flex`, `.gap-sm`.
