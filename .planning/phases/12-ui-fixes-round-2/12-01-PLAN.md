  <action>
Make these specific changes to static/style.css:

**FIX-01 — Ghost card opacity:** Find `.card--empty` (currently `opacity: 0.35`). Change to `opacity: 0.55`. This makes the greyed-out "not found" cards readable while still visually distinct from active cards.

**FIX-02 — Consistent price sizing:** The issue is in index.html (Task 2), but we also need a CSS class. Add a new utility class after the `.text-price` definition (around line 533):
```css
.card-price { font-size: var(--text-price); font-weight: 600; line-height: 1.2; }
```
This gives all card prices the same size regardless of deal status.

**FIX-07 — Scrollbar:** The scrollbar CSS already exists in section 19 (lines 812-833) with both webkit and Firefox rules. The issue is likely that Firefox's `* { scrollbar-width: thin; scrollbar-color: ... }` selector is too broad and may conflict. Change the Firefox scrollbar rule from `*` to `html` to target the document scrollbar specifically:
```css
html {
  scrollbar-width: thin;
  scrollbar-color: #444444 #1a1a1a;
}
```
Remove the `* { scrollbar-width ... }` rule.

**FIX-08 — Card titles too small:** Find `--text-title: 16px;` in the CSS custom properties (line 87). Change to `--text-title: 18px;`. This makes card titles larger while keeping them subordinate to artwork.

**FIX-03 — Font overhaul (CSS part):** Add an `@font-face` rule for Inter (variable weight woff2). Add it after the existing Bodoni Moda `@font-face` block:
```css
@font-face {
  font-family: "Inter";
  src: url("fonts/Inter-VariableFont_opsz,wght.woff2") format("woff2");
  font-weight: 100 900;
  font-style: normal;
  font-display: swap;
}
```
Then change `--font-sans` from the system font stack to:
```css
--font-sans: "Inter", -apple-system, BlinkMacSystemFont, "Segoe UI", Helvetica, Arial, sans-serif;
```
This makes Inter the primary UI font with system fallbacks. The `--font-display` (Bodoni Moda) for the CRATE logo remains unchanged.

**Font file:** Download the Inter variable woff2 font file. Run:
```bash
curl -L "https://github.com/rsms/inter/releases/download/v4.1/Inter-4.1.zip" -o /tmp/inter.zip
unzip -o /tmp/inter.zip -d /tmp/inter
cp /tmp/inter/Inter-VariableFont_opsz,wght.woff2 static/fonts/Inter-VariableFont_opsz,wght.woff2
```
If the exact path inside the zip differs, find the woff2 variable font file and copy it.

<link rel="preload" href="/static/fonts/Inter-VariableFont_opsz,wght.woff2" as="font" type="font/woff2" crossorigin>

**FIX-02 — Consistent price sizing in index.html:** In the card body section (around lines 185-193), the price display uses different classes/styles depending on deal status. Currently:
- Deal: `<span class="text-price price-best">` (28px)
- Non-deal: `<span class="price-best" style="font-size: var(--text-body);">` (16px)

Change both to use the same `card-price` class:
```html
{% if item.best_price is not none %}
  <span class="card-price price-best">${{ '%.2f'|format(item.best_price) }}</span>
{% elif not is_scanning %}
  <span class="text-faint" style="font-size: var(--text-label);">No listings found yet</span>
{% endif %}
```
This removes the `{% if has_deal %}` / `{% else %}` branching for price display — all prices use `card-price` class (which maps to `--text-price: 28px`).
