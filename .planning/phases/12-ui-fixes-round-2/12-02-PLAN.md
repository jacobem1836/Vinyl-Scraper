**Root cause:** HTML checkboxes don't send any value when unchecked. FastAPI's `Form(True)` default means unchecked = absent = defaults to True. The item is created with `notify_email=True` even when the user unchecked the box.

**Fix approach:** Use a hidden field pattern. Before each checkbox in the HTML forms, add a hidden input with the same name and value `""` (empty string). When the checkbox is checked, the checkbox value (`on`) overrides the hidden field. When unchecked, only the hidden field's empty string is sent, which FastAPI parses as falsy.

**Changes to app/routers/wishlist.py:**

1. Find the `add_wishlist_item` route (line ~95). Change the `notify_email` parameter from:
```python
notify_email: bool = Form(True),
```
to:
```python
notify_email: str = Form(""),
```
Then in the function body, convert to bool before using:
```python
notify_email_bool = notify_email.lower() in ("on", "true", "1", "yes")
```
Use `notify_email_bool` wherever `notify_email` was used in the item creation (line ~109):
```python
notify_email=notify_email_bool,
```

2. Find the `edit_wishlist_item` route (line ~125). Apply the same pattern:
Change:
```python
notify_email: bool = Form(False),
```
to:
```python
notify_email: str = Form(""),
```
And convert:
```python
notify_email_bool = notify_email.lower() in ("on", "true", "1", "yes")
```
Use `notify_email_bool` on line ~142:
```python
item.notify_email = notify_email_bool
```

**Changes to templates/index.html and templates/item_detail.html:**

In BOTH files, find each checkbox input for `notify_email` and add a hidden field BEFORE it:
```html
<input type="hidden" name="notify_email" value="" />
<input type="checkbox" name="notify_email" ... />
```

In index.html, this appears twice (add modal ~line 81, edit modal ~line 130).
In item_detail.html, this appears once (edit modal ~line 224).
