## Component Inventory

New components required for this phase (all CSS-only, no JS framework):

### 1. Typeahead Input Wrapper (`.typeahead-wrapper`)

- `position: relative` container around the query `<input>` and the dropdown
- Dropdown absolutely positioned below input, full width, z-index 60 (above modal z-index 50)
- Input reuses existing `.form-input` class unchanged

### 2. Typeahead Dropdown (`.typeahead-dropdown`)

- `position: absolute; top: 100%; left: 0; right: 0`
- `background: var(--color-surface-alt)`
- `border: 1px solid var(--color-border)`
- `border-radius: var(--radius-input)` (2px — matches input border radius)
- `z-index: 60`
- `max-height: 260px; overflow-y: auto`
- Hidden by default (`display: none`); shown via JS by toggling `.typeahead-dropdown--open` class

### 3. Typeahead Result Row (`.typeahead-row`)

- `display: flex; align-items: center; gap: var(--space-sm)`
- `padding: var(--space-sm) var(--space-md)`
- `cursor: pointer`
- `border-bottom: 1px solid var(--color-border-muted)` — last child: no border
- Hover/keyboard-focus state: `background: var(--color-surface)` + left accent border: `border-left: 2px solid var(--color-accent)`
- Active/selected state: same as hover

### 4. Typeahead Cover Thumb (`.typeahead-thumb`)

- `width: 40px; height: 40px`
- `object-fit: cover`
- `flex-shrink: 0`
- `border-radius: 0` — matches CRATE's zero-radius card aesthetic
- Fallback: vinyl placeholder SVG if no thumb available

### 5. Typeahead Row Text Block

- Title: `<span class="text-sm">` with `color: var(--color-text)`
- Subtitle: `<span class="text-sm text-muted">` — format: `{artist} · {year}` — `{year}` omitted if not present

### 6. Pinned Release Badge (`.typeahead-pin-badge`)

- Extends `.badge.badge-album` — uses album badge colors
- Contains: release title + year in parentheses, then a `×` clear button
- Clear button: plain `<button>` with `type="button"`, `color: var(--color-text-muted)`, hover: `color: var(--color-destructive)`
- Badge displayed above the typeahead input when a release is pinned (decision D-04)
- Layout: `display: flex; align-items: center; gap: var(--space-xs)`

### 7. Locked Query Input State

- When a release is pinned, query input gets attribute `readonly` and class `.form-input--locked`
- `.form-input--locked`: `opacity: 0.5; cursor: not-allowed; background: var(--color-surface)`
- Focus ring suppressed when locked: `:focus` outline remains `none`, `box-shadow` removed
- Tooltip/title attribute: `"Clear the pinned release above to edit this field"` (native browser tooltip)

### 8. Typeahead Loading Indicator

- A 14px `.spinner` (already defined in `static/style.css`) shown inline at the right edge of the input wrapper while a fetch is in-flight
- Positioned: `position: absolute; right: var(--space-md); top: 50%; transform: translateY(-50%)`
- Removed on response arrival (success or error)

### 9. Typeahead Empty / Error State

- Empty dropdown with single non-interactive row: `No results found for "{query}"` — `color: var(--color-text-faint); font-size: var(--text-sm); padding: var(--space-sm) var(--space-md)`
- Network error: same row format but copy: `Search unavailable. Try again.` — `color: var(--color-text-faint)`
- No results and error states still render the dropdown container (so the input doesn't visually jump)

## Interaction Contract

### Debounce

- 300ms minimum debounce on input keystrokes before firing the search request (requirement TYPE-04)
- Implementation: `setTimeout` / `clearTimeout` pattern in vanilla JS
- AbortController: previous in-flight fetch cancelled on new keystroke via `AbortController.abort()`

### Dropdown Trigger

- Dropdown activates only when `type` select value is `"album"` or `"subject"` (decision D-01)
- When `type` is `"artist"` or `"label"`: query input behaves as plain text, no dropdown, no spinner
- Minimum query length to trigger: 2 characters (prevents pointless single-char searches)

### Keyboard Navigation

- `ArrowDown` / `ArrowUp`: move highlighted row (wraps at ends)
- `Enter`: select highlighted row (same as click)
- `Escape`: close dropdown without selecting; return focus to input
- `Tab`: close dropdown (focus moves naturally to next field)
- Highlighted row tracked in JS, reflected by `.typeahead-row--active` class on the row

### Selection

- On select: autofill query input with release title, set hidden input `discogs_release_id` value, show pinned badge, lock query input, close dropdown
- Pinned badge shows: `{title} ({year})` — year omitted if absent

### Clear Badge

- Clicking `×` on the pinned badge: remove badge, clear hidden `discogs_release_id` input, unlock query input, clear query input value, focus query input
- Does not require confirmation — reversible action (user can re-search)

### Edit Modal Pre-State (D-04)

- When `openEdit()` is called with a `data-discogs-release-id` attribute on the trigger, the modal opens with the badge already rendered and the query input locked
- When `data-discogs-release-id` is absent or `null`, modal opens in default state (no badge, unlocked input)

### Rate Limit Protection

- Debounce (300ms) combined with AbortController prevents request flooding
- Maximum 5 results returned from `GET /api/discogs/search?q=` endpoint (requirement TYPE-04)
- Spinner shown during fetch — user has visual feedback that a request is pending

## Accessibility Contract

- Typeahead input: `role="combobox"`, `aria-expanded="true|false"`, `aria-haspopup="listbox"`, `aria-autocomplete="list"`, `aria-controls="typeahead-listbox"`
- Dropdown container: `role="listbox"`, `id="typeahead-listbox"` (or `id="edit-typeahead-listbox"` for the edit modal)
- Each result row: `role="option"`, `aria-selected="true|false"`
- Active row: `aria-activedescendant` on input updated to active row `id` on keyboard move
- Clear badge button: `type="button"`, `aria-label="Clear pinned release"`
- Spinner: `aria-label="Searching Discogs..."`, `aria-live="polite"` on wrapper to announce loading state changes
- Locked input: `readonly` attribute + `title="Clear the pinned release above to edit this field"`
