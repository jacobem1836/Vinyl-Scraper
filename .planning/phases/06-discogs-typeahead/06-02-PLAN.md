**Create `static/typeahead.js`** — Self-contained typeahead module with all shared functions. This file must be usable from both index.html and item_detail.html. Implement:

1. **Shared state/config:**
   - `const DEBOUNCE_MS = 300;`
   - `const MIN_QUERY_LEN = 2;`
   - A state map keyed by prefix: `const state = {};`

2. **`window.initTypeahead = function(prefix, typeSelectEl)`** — Initialize typeahead for a given prefix ('add' or 'edit'). Parameters: prefix string, and a reference to the type `<select>` element for this modal. Sets up:
   - State object: `state[prefix] = { timer: null, controller: null, activeIndex: -1, results: [] }`
   - References to elements by ID convention: `${prefix}Query` (input), `${prefix}PinBadge` (badge container), `${prefix}PinText` (badge text span), `${prefix}DiscogsReleaseId` (hidden input), `${prefix}Spinner` (spinner), `${prefix}-typeahead-listbox` (dropdown)
   - Input event listener on the query input with debounce + AbortController pattern
   - Keydown listener on the query input for ArrowUp, ArrowDown, Enter, Escape, Tab
   - Change listener on typeSelectEl: when type changes to "artist" or "label", close dropdown and clear pin; when type is "album" or "subject", typeahead is active

3. **Debounced fetch (internal to initTypeahead):**
   - On input event, clear previous timer (`clearTimeout`), abort previous controller (`controller.abort()`)
   - If input is readonly (locked), return immediately
   - If query length < MIN_QUERY_LEN or type select value is not "album"/"subject" (per D-01): close dropdown, return
   - Set timer: after DEBOUNCE_MS, create new AbortController, show spinner, fetch `/api/discogs/search?q=${encodeURIComponent(query)}` with `{ signal: controller.signal }`
   - On success (200): parse JSON, store in `state[prefix].results`, render results, show dropdown, hide spinner
   - On AbortError: do nothing (expected during rapid typing)
   - On other error: show "Search unavailable. Try again." in dropdown, hide spinner

4. **Render results (internal helper):**
   - Clear listbox innerHTML
   - If results array is empty: insert `<div class="typeahead-empty">No results for "${query}"</div>`
   - For each result: create `<div class="typeahead-row" role="option" id="${prefix}-option-${i}" aria-selected="false" data-index="${i}">` containing:
     - `<img class="typeahead-thumb" src="/api/artwork?url=${encodeURIComponent(result.thumb)}" onerror="this.src='/static/vinyl-placeholder.svg'" alt="" />` — if no thumb, use placeholder directly
     - `<div>` with:
       - Title: `<span class="text-sm" style="color: var(--color-text)">${escapeHtml(result.title)}</span>`
       - Subtitle: `<span class="text-sm text-muted">${escapeHtml(result.artist)}${year ? ' \u00b7 ' + year : ''}</span>` — year omitted if null/empty/"0"
   - Click handler on each row: call `window.selectResult(prefix, result)`
   - Mouseover on each row: update activeIndex, update visual highlight
   - Add `.typeahead-dropdown--open` class to show dropdown
   - Set `aria-expanded="true"` on input

5. **Keyboard navigation (per TYPE-02):**
   - ArrowDown: increment activeIndex (wrap to 0 past end), update `.typeahead-row--active` class, set `aria-activedescendant` on input to active row id. Prevent default scroll.
   - ArrowUp: decrement activeIndex (wrap to last past start). Prevent default.
   - Enter: if dropdown is open and activeIndex >= 0, select that result (`selectResult`), prevent default (prevent form submit).
   - Escape: close dropdown, reset activeIndex to -1
   - Tab: close dropdown

6. **`window.selectResult = function(prefix, result)`** — Per D-05:
   - Set hidden input: `document.getElementById(prefix + 'DiscogsReleaseId').value = result.release_id`
   - Set query input value to `result.title`
   - Lock query input: add class `form-input--locked`, set `readOnly = true`, set placeholder to `"Release pinned \u2014 clear badge to change"`, set title attribute to `"Clear the pinned release above to edit this field"`
   - Show pin badge: remove `hidden` class from badge element, set text span to `${result.title}${result.year && result.year !== '0' ? ' (' + result.year + ')' : ''}`
   - Close dropdown (remove `--open` class, set `aria-expanded="false"`)
   - Reset activeIndex to -1

7. **`window.clearPin = function(prefix)`** — Per D-05:
   - Clear hidden input value to empty string
   - Remove `form-input--locked` class from query input, set `readOnly = false`
   - Set placeholder back to `"Search Discogs..."`
   - Remove title attribute
   - Clear query input value
   - Hide pin badge (add `hidden` class)
   - Focus query input

8. **`window.setPin = function(prefix, releaseId, displayText)`** — Helper for edit modal pre-state (D-04):
   - Set hidden input value to releaseId
   - Lock query input (add `form-input--locked`, set `readOnly = true`, set placeholder, set title)
   - Show pin badge with displayText
   - Does NOT set query input value (the existing openEdit handler already does that)

9. **Close dropdown helper (internal):** Remove `--open` class, set `aria-expanded="false"`, reset activeIndex to -1

10. **`escapeHtml` helper (internal):** Escape `&`, `<`, `>`, `"` characters to prevent XSS from Discogs data (T-06-06 mitigation).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser JS -> /api/discogs/search | User-typed query sent to backend, proxied to Discogs |
| browser form POST -> backend | Hidden discogs_release_id submitted -- integer or empty |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-05 | Denial of Service | Typeahead rapid fire | mitigate | 300ms debounce in JS + AbortController cancels in-flight requests. Server-side Discogs rate limit (60/min) is backstop. Max 5 results per query. |
| T-06-06 | Spoofing | XSS via typeahead results | mitigate | Results rendered via textContent/escapeHtml (not raw innerHTML for text). Image URLs proxied through /api/artwork. No raw HTML injection from Discogs data. |
| T-06-07 | Tampering | Hidden input discogs_release_id | accept | User can modify value via devtools; single-user app, no privilege escalation. Worst case: wrong Discogs release linked to their own item. |
</threat_model>
