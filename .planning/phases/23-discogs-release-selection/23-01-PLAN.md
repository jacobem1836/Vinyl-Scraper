# Plan 23-01: Discogs Release Pinning

## Goal

Deliver the complete release pinning feature: search API endpoint, pin/unpin POST endpoint, and the full modal UI with search, selection, confirm, clear, and pin status display.

## Context

Users currently get auto-matched Discogs results which can be wrong (wrong pressing, wrong artwork). The `discogs_release_id` column already exists on `WishlistItem` and the scanner already respects it when set. What is missing is: (1) a way for users to search releases, (2) a way to pin/unpin a release with artwork update, and (3) the UI to drive this flow.

<objective>
Implement Discogs release pinning: a dedicated search endpoint, a pin/unpin endpoint that updates both `discogs_release_id` and `artwork_url`, and the full pin modal UI on item detail with search, results list, confirm, clear pin, and pin status label.

Purpose: Let users fix wrong artwork and scan mismatches by manually selecting the correct Discogs release.
Output: Two new endpoints in wishlist.py, updated item_detail.html with pin button + modal + status label.
</objective>
