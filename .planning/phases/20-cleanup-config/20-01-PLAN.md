# Plan 20-01: Remove dead Clarity adapter and harden eBay config

## Goal
Delete the dead Clarity Records adapter and ensure eBay credentials produce a clear diagnostic when missing in production.

## must_haves
- [ ] `app/services/clarity.py` does not exist
- [ ] `app/services/adapter.py` has no import of `clarity` and no registry entry for it
- [ ] `app/services/ebay.py` prints a warning when `ebay_app_id` or `ebay_cert_id` is missing
- [ ] eBay adapter still returns listings when credentials are present (no functional change)

## Tasks

<task id="20-01-T1">
<title>Remove Clarity adapter and registry entry</title>
<read_first>
- app/services/clarity.py — the file to delete
- app/services/adapter.py — contains the import and registry entry to remove
</read_first>
<action>
1. Delete the file `app/services/clarity.py` entirely.

2. In `app/services/adapter.py`, change the import line from:
   ```python
   from app.services import bandcamp, clarity, discogs, discrepancy, ebay, juno, shopify
   ```
   to:
   ```python
   from app.services import bandcamp, discogs, discrepancy, ebay, juno, shopify
   ```

3. In `app/services/adapter.py`, remove the entire registry entry line:
   ```python
   {"name": "clarity", "fn": clarity.search_and_get_listings, "enabled": False},  # Clarity site unreachable during implementation -- enable once confirmed
   ```

No other files reference clarity, so no further changes are needed.
</action>
<acceptance_criteria>
- `ls app/services/clarity.py` returns "No such file or directory"
- `grep -r "clarity" app/services/adapter.py` returns no matches
- `grep -r "clarity" app/` returns no matches
- `python -c "from app.services.adapter import ADAPTER_REGISTRY; print(len(ADAPTER_REGISTRY))"` prints `6` (was 7)
- `python -c "from app.services.adapter import ADAPTER_REGISTRY; print([a['name'] for a in ADAPTER_REGISTRY])"` prints `['discogs', 'shopify', 'ebay', 'discrepancy', 'juno', 'bandcamp']`
</acceptance_criteria>
</task>

<task id="20-01-T2">
<title>Add diagnostic logging for missing eBay credentials</title>
<read_first>
- app/services/ebay.py — current eBay adapter; lines 46-48 silently return [] when credentials missing
- app/config.py — confirms ebay_app_id and ebay_cert_id are Optional[str] with None defaults
</read_first>
<action>
In `app/services/ebay.py`, update the credential check in `search_and_get_listings` (lines 47-48) from:

```python
    if not settings.ebay_app_id or not settings.ebay_cert_id:
        return []
```

to:

```python
    if not settings.ebay_app_id or not settings.ebay_cert_id:
        print("[eBay] Skipping — credentials not configured (set EBAY_APP_ID and EBAY_CERT_ID env vars)")
        return []
```

This is the only change. The eBay adapter already reads credentials correctly from env vars via pydantic-settings. The diagnostic log ensures operators can see why eBay returns no results in production logs if the Railway env vars are not set.
</action>
<acceptance_criteria>
- `grep 'credentials not configured' app/services/ebay.py` returns exactly one match
- `grep 'EBAY_APP_ID and EBAY_CERT_ID' app/services/ebay.py` returns exactly one match
- `grep -c 'print.*eBay.*credentials' app/services/ebay.py` returns `1`
- The function still returns `[]` after the print (no functional change to existing behavior)
</acceptance_criteria>
</task>
