## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| `--space-xs` | 4px | Icon gaps, badge padding, inline micro-gaps |
| `--space-sm` | 8px | Compact element spacing, button icon gap |
| `--space-md` | 16px | Default padding, card inner padding, form field gap |
| `--space-lg` | 24px | Section padding, card grid gap |
| `--space-xl` | 32px | Major section vertical rhythm |
| `--space-2xl` | 48px | Page top/bottom padding |
| `--space-3xl` | 64px | Large empty state padding |

Exceptions:
- Touch targets (scan button, CTA button): minimum 40px height (close to 44px WCAG guideline; 40px used for visual compactness, acceptable for desktop-first tool). Source: REQUIREMENTS.md — out of scope note "Mobile app: iOS Shortcut handles mobile adds; web is desktop-first."
- Nav bar height: 64px (matches existing `h-16`; preserve as-is).

## Typography

All font sizes are CSS custom property tokens.

| Token | Role | Size | Weight | Line Height | Usage |
|-------|------|------|--------|-------------|-------|
| `--text-sm` | Label/Meta | 14px | 400 | 1.5 | Form labels, table headers, badge text, card metadata, nav links, timestamps, found-at dates, placeholder copy, faint labels |
| `--text-body` | Body | 16px | 400 | 1.5 | Default paragraph text, modal body, listing row text |
| `--text-heading` | Heading | 24px | 600 | 1.2 | Item detail page H1, card section heading |

Additional display weight for price figures:

| Token | Role | Size | Weight | Line Height | Usage |
|-------|------|------|--------|-------------|-------|
| `--text-price` | Price display | 20px | 600 | 1.2 | AUD total in detail page listing cards |

**Weights used:** 400 (regular) and 600 (semibold) only. No other weights.

Font declaration:
```css
--font-sans: -apple-system, BlinkMacSystemFont, "Segoe UI", Helvetica, Arial, sans-serif;
```

Source: defaults (Claude's discretion per CONTEXT.md). Consistent with existing Tailwind utility usage in templates.

## Color — 60/30/10 Split

| Role | Hex / Token | Coverage | Usage |
|------|-------------|----------|-------|
| Dominant (60%) | `#0f172a` (`--color-bg`) | Page background, nav bar, table header rows, modal backdrops, input backgrounds | Full-bleed page surface |
| Secondary (30%) | `#1e293b` (`--color-surface`) | Cards (dashboard grid), panels (scan progress, stats bar), modal panels, listing table container | All contained surfaces |
| Accent (10%) | `#f59e0b` (`--color-accent`) | See reserved-for list below | Intentional attention elements only |
| Destructive | `#f87171` (`--color-destructive`) | Delete button text and border; confirm copy | Destructive actions exclusively |

**Accent reserved for (explicit list):**
1. Nav bar brand wordmark ("Vinyl Wishlist")
2. Primary CTA buttons background (Add Item, Scan All, Scan Now, Add to Wishlist, Save Changes)
3. Best price value in card and table (`price-best` class)
4. "View Deals →" link on dashboard cards
5. "← Back to Dashboard" link on detail page
6. Spinner border (top segment — existing pattern)
7. Pulse border animation on scanning cards
8. Form input focus ring (`ring-2` equivalent)
9. Notify-below-pct label highlight in detail header

Accent is NOT used for: general text, borders, badges, table row hover, secondary buttons.

Source: CONTEXT.md D-02, D-05, D-16; confirmed from existing `base.html` and `index.html`.

## Layout Contracts

### Dashboard Grid (UI-02)

```
Mobile  (<768px):  1 column
Tablet  (≥768px):  2 columns
Desktop (≥1280px): 4 columns
```

- Grid gap: `--space-lg` (24px)
- Card aspect: artwork fills card face at 1:1 aspect ratio via `aspect-ratio: 1 / 1` + `object-fit: cover`
- Below artwork: item title at `--text-sm` (14px, weight 600) and best AUD price at `--text-sm` (14px, weight 400 amber)
- No listing count on card. No last-scanned timestamp on card. Source: CONTEXT.md D-04.
- Card click target: entire card navigates to `item_detail.html`. Source: CONTEXT.md D-05.
- Max content width: 1280px, centred, `padding-inline: --space-md` on small screens.

Source: Claude's discretion (CONTEXT.md allows grid column count and card sizing).

### Card Hover/Focus States

- Hover: `transform: translateY(-2px)` + `box-shadow: 0 8px 24px rgba(0,0,0,0.4)` — 200ms ease.
- Focus (keyboard): `outline: 2px solid var(--color-accent)` offset 2px.
- Scanning card: existing `pulse-border` keyframe animation preserved, restyled using `--color-accent`.

Source: Claude's discretion (CONTEXT.md allows hover/focus states).

### Artwork Proxy (UI-03)

- Artwork src: `<img src="/api/artwork?url={{ item.artwork_url }}" …>` — never hotlink Discogs CDN. Source: CONTEXT.md D-09.
- Placeholder: `static/vinyl-placeholder.svg` served as a static file. Dark vinyl SVG, fits aesthetic. Source: CONTEXT.md D-10.
- Fallback: `onerror="this.src='/static/vinyl-placeholder.svg'"` on every `<img>` artwork element.

### Item Detail Page

- Header: artwork thumbnail (120px × 120px) floated left of item title + meta. Source: design extension of D-03.
- "Best Deals" section: card grid 1/2/3 columns (mobile/md/xl).
- Listing row in "All Listings" table: AUD total prominent (`--text-price`, amber), original-currency amount below in `--text-sm` muted. Source: CONTEXT.md D-11.
- FX rate label: not shown in UI. Source: CONTEXT.md D-11.

### Scan Polling UX

Floating pill + progress panel from Phase 1 preserved and restyled. No structural changes to JS logic. Source: CONTEXT.md D-06.

## Interaction Contracts

### Add Item Modal
- Triggered by: "Add Item" button in nav bar.
- Fields: query (text, required), type (select: album/artist/label/subject), notes (text, optional), notify_below_pct (number, 1–90, default 20), notify_email (checkbox, default checked).
- Submit: `POST /wishlist/add` (form action, Jinja2 page reload).
- Cancel: close modal, no changes.
- Backdrop click: close modal.

### Edit Item Modal
- Triggered by: "Edit" button on dashboard card.
- Same fields as Add Item, pre-populated via `data-*` attributes.
- Submit: `POST /wishlist/{id}/edit`.

### Delete Item
- Triggered by: "Delete" button on item detail page.
- Method: `POST /wishlist/{id}/delete` (form submit).
- Confirmation: `onsubmit="return confirm('Remove this item from your wishlist?')"`.
- No separate confirmation modal — inline confirm() is sufficient for personal tool. Source: existing pattern retained.

### Scan Trigger
- Dashboard: "Scan All" calls `startScan()` with no argument.
- Detail: "Scan Now" calls `startScan({item.id})`.
- Both show floating scan pill (preserved from Phase 1).
