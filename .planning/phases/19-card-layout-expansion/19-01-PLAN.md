<objective>
Expand wishlist cards to fill more screen width and feel more spacious by making three
targeted CSS edits and one HTML attribute removal.

The 4-column breakpoint at 1280px is removed so large screens top out at 3 columns
(the existing 1024px rule). Gap between cards is widened from 8px to 16px. Container
and nav side padding are reduced from 16px to 12px, giving more horizontal room to the
grid. The negative-margin inline bleed on the card-grid section element is removed
because with only 12px of container padding the bleed would leave an unusably thin 4px
gap to the viewport edge.

Purpose: Cards feel undersized and cramped at 1440px wide; removing the 4-col tier and
tightening side padding makes better use of the available canvas without changing any
backend logic or adding new UI components.

Output: Modified static/style.css (3 property changes, 4 lines removed) and modified
templates/index.html (1 inline style attribute removed).
</objective>
