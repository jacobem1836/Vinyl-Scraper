### 60/30/10 Contract

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#000000` | Page background (`--color-bg`), card surfaces (`--color-surface`), form inputs / nav / typeahead (`--color-surface-alt`) |
| Secondary (30%) | `#222222` (border) | Visual separation between surfaces — `--color-border` provides all elevation; no secondary fill surface used (D-02) |
| Accent (10%) | `#ffffff` | CTA button fill, primary interactive text, focus rings |
| Destructive | `#c46060` | Delete/remove actions only — never decorative |

Accent reserved for: CTA button background (`--color-accent-cta`), focused/active interactive states, primary link text where emphasis is required. Never used for decorative borders, backgrounds, or disabled states.

## Scrollbar Contract

Full rework of section 19 in `static/style.css`. All existing scrollbar rules are replaced.

### WebKit (Chrome, Safari, Edge)

Apply to: global (`html` / `:root`), `.typeahead-dropdown`, `.table-container`

| Property | Value |
|----------|-------|
| Width | 4px |
| Track background | transparent |
| Thumb at rest | `rgba(255,255,255,0.2)` |
| Thumb on `:hover` | `rgba(255,255,255,0.4)` |
| Thumb `border-radius` | 0px (matches `--radius-card: 0px` — no rounding) |

Source: D-04.

### Firefox

```css
html {
  scrollbar-width: thin;
  scrollbar-color: rgba(255,255,255,0.2) transparent;
}
```

Source: D-06.

### Scope

Apply WebKit rules to all three contexts:
1. Global page scrollbar (`::-webkit-scrollbar` selectors on `:root` or globally)
2. `.typeahead-dropdown` (has `overflow-y: auto`) — D-05
3. `.table-container` (has `overflow-x: auto`) — D-05

For `.typeahead-dropdown` and `.table-container`, use scoped selectors:
```css
.typeahead-dropdown::-webkit-scrollbar { ... }
.table-container::-webkit-scrollbar { ... }
```
