    3. Section 19 scrollbars reworked to 4px transparent-track / `rgba(255,255,255,0.2)` thumb / `0.4` hover, scoped globally + `.typeahead-dropdown` + `.table-container`, with matching Firefox `scrollbar-color`
  </what-built>
  <how-to-verify>
    Open http://localhost:8000 in Chrome (or Safari). Walk through the following and confirm each:

    **VIS-01 — True black (#000):**
    1. Dashboard page: inspect page background — should read as pure black, no warm/brown cast. Use DevTools colour picker on an empty area → expect `rgb(0, 0, 0)`.
    2. A card: background of the card itself (not just the page) should also be `#000` — card separation comes from `#222` border only, not fill.
    3. Click into item detail. Page + card surfaces should all be `#000`.
    4. Click "Add to Wishlist". Modal backdrop + modal panel background should be `#000`.
    5. Typeahead dropdown and nav bar → also `#000`.

    **VIS-02 — Custom scrollbars:**
    6. Scroll the dashboard (or any page with a long wishlist). Scrollbar should be:
       - ~4px wide (not ~12–16px browser default)
       - Track invisible (no visible rail)
       - Thumb a translucent white, barely there at rest
       - Thumb brightens noticeably when you hover over it
    7. In the add-item typeahead dropdown, type enough to generate many results. The internal dropdown scrollbar should match global styling (4px, translucent white thumb).
    8. On item detail, if the listings `.table-container` horizontally overflows, its scrollbar should match.
    9. In Firefox (if available): scrollbar should be `thin` with matching translucent-white thumb on transparent track.

    **VIS-03 — Warner Music–inspired:**
    10. Compare against Warner Music reference (stark white-on-black, no card chrome, editorial restraint). Overall page should read as flat, editorial, restrained — NOT glossy, NOT warm, NOT decorative.
    11. Skeleton pulse: hard-refresh a page to catch the artwork skeleton mid-load. The pulse should be visible but subtle (dark grey on black, not invisible). If it's completely invisible, report back so Claude can raise `--color-border-muted` / adjust pulse values per D-03 discretion clause.
    12. `--color-border-muted` check: are there any borders / dividers that have become invisible against `#000`? (Common candidates: subtle separator lines, muted input borders.) If yes, flag them so Claude can raise `#111111` → `#1a1a1a` per CONTEXT.md §Claude's Discretion.
  </how-to-verify>
