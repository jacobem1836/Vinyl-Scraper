# Plan 2: Adapter Registry — Source-Agnostic Scanner

**Phase:** 1 — Infrastructure
**Goal:** Scanner iterates a central registry instead of hard-coding sources; adding a new source = one adapter file + one registry entry
**Requires:** Plan 1 (scanner.py is modified in Plan 1; this plan builds on top of those changes)
**Requirements:** SRC-06

## Tasks

### Task 1: Create the adapter registry with ListingDict type

**File(s):** `app/services/adapter.py`
**What:** Create a module that defines the standardized listing dict as a TypedDict and the registry as a list of dicts with `name`, `fn`, and `enabled` fields.
**How:**
- Create `app/services/adapter.py` with:
  - `ListingDict(TypedDict, total=False)` containing all keys currently used by discogs.py and shopify.py: `source` (str, required), `title` (str, required), `url` (str, required), `price` (float | None), `currency` (str), `condition` (str | None), `seller` (str | None), `ships_from` (str | None), `is_in_stock` (bool)
  - Type alias: `AdapterFn = Callable[[str, str], Awaitable[list[dict]]]` — matches the existing `search_and_get_listings(query, item_type)` signature
  - `ADAPTER_REGISTRY: list[dict]` — list of dicts per D-10/D-11:
    ```python
    ADAPTER_REGISTRY = [
        {"name": "discogs", "fn": discogs.search_and_get_listings, "enabled": True},
        {"name": "shopify", "fn": shopify.search_and_get_listings, "enabled": True},
    ]
    ```
  - Import discogs and shopify at module level: `from app.services import discogs, shopify`
  - Helper function `get_enabled_adapters() -> list[dict]` that returns `[a for a in ADAPTER_REGISTRY if a["enabled"]]` (per D-12)
- The `enabled` flag on each entry supports future per-source toggling (per D-16) without overengineering now.

**Acceptance:** `from app.services.adapter import ADAPTER_REGISTRY, get_enabled_adapters, ListingDict` imports successfully. `get_enabled_adapters()` returns both discogs and shopify entries.

### Task 2: Refactor scanner.py to use the registry

**File(s):** `app/services/scanner.py`
**What:** Replace the hard-coded `discogs` and `shopify` imports and `asyncio.gather()` call in `scan_item()` with a loop over `get_enabled_adapters()`.
**How:**
1. Remove the direct imports of discogs and shopify (lines 7-8: `from app.services import discogs, shopify`)
2. Add: `from app.services.adapter import get_enabled_adapters`
3. In `scan_item()`, replace the current gather call (lines 11-14):
   ```python
   # OLD:
   discogs_results, shopify_results = await asyncio.gather(
       discogs.search_and_get_listings(item.query, item.type),
       shopify.search_and_get_listings(item.query, item.type),
   )
   all_results = discogs_results + shopify_results
   ```
   With:
   ```python
   # NEW (per D-12):
   adapters = get_enabled_adapters()
   results = await asyncio.gather(
       *[a["fn"](item.query, item.type) for a in adapters],
       return_exceptions=True,
   )
   all_results = []
   for i, r in enumerate(results):
       if isinstance(r, Exception):
           print(f"[Scanner] {adapters[i]['name']} error: {r}")
       else:
           all_results.extend(r)
   ```
4. Everything else in `scan_item()` stays the same — the dedup logic, listing creation, and commit flow are source-agnostic already.
5. The `scan_all_items()` function needs no changes — it calls `scan_item()` which now uses the registry internally.

**Acceptance:**
- `scan_item()` no longer imports `discogs` or `shopify` directly
- Adding a third adapter to `ADAPTER_REGISTRY` in `adapter.py` (e.g., a dummy `{"name": "test", "fn": some_fn, "enabled": True}`) causes `scan_item()` to call it automatically — scanner.py requires zero changes
- Disabling an adapter (`"enabled": False`) causes `scan_item()` to skip it
- Existing scan behavior is unchanged: discogs and shopify results still appear for items
