Steps:
1. Create directory `static/fonts/` if it does not exist.
2. Download Bodoni Moda Bold woff2 from Google Fonts. Use the Google Fonts CSS API to discover the woff2 URL:
   - Fetch `https://fonts.googleapis.com/css2?family=Bodoni+Moda:wght@700&display=block` with a User-Agent that triggers woff2 format (e.g., a modern Chrome UA string).
   - Extract the `url(...)` from the CSS response. It will point to a fonts.gstatic.com woff2 file.
   - Download that woff2 file and save it as `static/fonts/BodoniModa-Bold.woff2`.
3. Verify the file is a valid woff2 (first 4 bytes should be `wOF2` or file size > 5KB).

Weight 700 (Bold) is chosen because the current `.nav-brand` uses `font-weight: 600` — Bold (700) is the closest standard weight and will render crisply for the high-contrast Bodoni Moda face. The executor may optionally consider ExtraBold (800) if design tools suggest it during execution, but Bold is the default.

Claude's discretion: The executor MAY subset the font to uppercase Latin glyphs only (A-Z, 0-9, space) using pyftsubset or similar if available, to reduce file size. CRATE only uses 5 uppercase characters. This is optional — the full woff2 is acceptable.

Three changes across two files. All per locked decisions D-03 through D-05.

**1. static/style.css — Add @font-face declaration (before :root block)**

Insert the following BEFORE the existing `:root {` block (before line ~56 where section 1 begins), as the first content in section 1:

```css
@font-face {
  font-family: "Bodoni Moda";
  src: url("fonts/BodoniModa-Bold.woff2") format("woff2");
  font-weight: 700;
  font-style: normal;
  font-display: block;
}
```

Key details per D-04: `font-display: block` prevents FOUT on the brand mark. Per D-06: only woff2 src, no woff fallback.

**2. static/style.css — Add --font-display custom property in :root**

In the `:root` block, after the existing `--font-sans` declaration (currently line 81), add:

```css
  --font-display: "Bodoni Moda", var(--font-sans);
```

Per D-05: this keeps the design system clean with a CSS custom property and provides a system-font fallback chain.

**3. static/style.css — Update .nav-brand rule**

Change the `.nav-brand` rule (lines 176-183) to use the new font. Replace the existing rule with:

```css
.nav-brand {
  font-family: var(--font-display);
  font-size: var(--text-body);
  font-weight: 700;
  letter-spacing: 0.25em;
  text-transform: uppercase;
  line-height: 1;
  color: var(--color-accent);
}
```

Changes from current: added `font-family: var(--font-display);`, changed `font-weight: 600` to `font-weight: 700` to match the Bold weight loaded in @font-face. Per D-03: this is the ONLY selector that gets the display font.

**4. templates/base.html — Add preload link**

In the `<head>`, BEFORE the existing `<link rel="stylesheet" ...>` line, add:

```html
  <link rel="preload" href="/static/fonts/BodoniModa-Bold.woff2" as="font" type="font/woff2" crossorigin>
```

The `crossorigin` attribute is required for font preloads even when same-origin. This ensures the browser fetches the font early, complementing `font-display: block` for zero-FOUT brand rendering.
