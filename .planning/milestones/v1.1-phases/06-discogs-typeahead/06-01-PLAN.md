1. **app/services/discogs.py** — Add a new `async def typeahead_search(query: str, max_results: int = 5) -> list[dict]` function:
   - Calls `GET /database/search` with params `q=query, type=release, format=Vinyl, per_page=max_results` (reuse `_get_headers()`, `BASE_URL`)
   - Returns a list of dicts, each with: `{"release_id": int, "title": str, "artist": str, "year": str|None, "thumb": str|None}`
   - Parse results: `title` field from Discogs contains "Artist - Title" format. Split on " - " to extract artist and title. If no separator, use full string as title and empty string as artist.

   - `thumb` from result's `thumb` or `cover_image` field
   - Wrap in try/except, return empty list on error (log with `print(f"[Discogs] Typeahead error: {e}")`)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> /api/discogs/search | Untrusted user input passed as search query to Discogs API |
| browser -> form POST | discogs_release_id from hidden input — user could tamper with value |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Injection | /api/discogs/search q param | mitigate | Query is passed to Discogs API as URL param via httpx (auto-encoded); no SQL or shell injection vector. Min 2 char check prevents empty queries. |
| T-06-02 | Denial of Service | /api/discogs/search | mitigate | 300ms client-side debounce + AbortController (Plan 02); max 5 results cap; single API call per request. Discogs own rate limit (60/min) is the backstop. |
| T-06-03 | Tampering | discogs_release_id form field | accept | User can set any integer; worst case is linking to wrong Discogs release (their own data, single-user app). No privilege escalation possible. |
| T-06-04 | Information Disclosure | /api/discogs/search response | accept | Returns public Discogs catalog data (titles, years, thumbs). No private data exposed. Endpoint has no auth — acceptable for single-user tool. |
</threat_model>
