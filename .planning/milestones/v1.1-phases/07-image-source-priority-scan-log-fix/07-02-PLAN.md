logEl.innerHTML = s.log.slice().reverse().map(function(entry) {

Update to include the type label from `entry.type` (now provided by scan_status per Plan 01). Change to:

```javascript
logEl.innerHTML = s.log.slice().reverse().map(function(entry) {
  var typeLabel = entry.type ? ' <span class="text-faint">(' + escHtml(entry.type) + ')</span>' : '';
  return '<div class="text-sm text-muted">' + escHtml(entry.query) + typeLabel + ' — <span class="text-success">+' + entry.new_listings + '</span></div>';
}).join('');
```

This renders as e.g. `Radiohead (album) — +3` in the scan log. The type label uses `text-faint` class (same as the active scan indicator on line 210 which already shows the type). If `entry.type` is empty or missing (backwards compat), no type label is shown.
