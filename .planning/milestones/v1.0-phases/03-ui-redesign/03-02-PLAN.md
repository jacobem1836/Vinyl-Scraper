**app/routers/wishlist.py** — Three changes:

1. Add import at top: `from fastapi.responses import RedirectResponse, StreamingResponse` (StreamingResponse is new, RedirectResponse already imported — just add StreamingResponse to the existing import). Also add `import httpx` at top.

2. Add `"artwork_url": item.artwork_url,` to the `_enrich_item` return dict, after `"is_active": item.is_active,` (per Pitfall 6 in RESEARCH.md — new ORM columns do not auto-appear in manually built dicts).

3. Add artwork proxy endpoint on `web_router` (NOT api_router — no API key needed for image serving):
```python
@web_router.get("/api/artwork")
async def proxy_artwork(url: str = ""):
    if not url:
        raise HTTPException(status_code=400, detail="url required")
    try:
        async with httpx.AsyncClient(timeout=10.0) as client:
            r = await client.get(url, headers={"User-Agent": "VinylWishlist/1.0"})
            if r.status_code != 200:
                raise HTTPException(status_code=502, detail="upstream error")
            return StreamingResponse(
                r.aiter_bytes(),
                media_type=r.headers.get("content-type", "image/jpeg"),
                headers={"Cache-Control": "public, max-age=86400"},
            )
    except httpx.HTTPError:
        raise HTTPException(status_code=502, detail="artwork fetch failed")
```
Note: Cache-Control max-age=86400 per Pitfall 5 recommendation for browser caching.
