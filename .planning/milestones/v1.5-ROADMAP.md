# Milestone v1.5: Coverage & Sources

**Status:** ✅ SHIPPED 2026-04-26
**Phases:** 25–27
**Total Plans:** 4

## Overview

Expanded the scraper's source coverage from a 6-store Shopify list and disabled eBay to a 12-store Shopify list, working eBay integration, and a new BigCommerce HTML scraper for Clarity Records. This milestone is purely additive — no data model changes, no UI changes.

## Phases

### Phase 25: eBay Credentials

**Goal**: Wire eBay credentials and verify adapter returns results
**Depends on**: Phase 20
**Requirements**: SRC-06
**Plans**: 1 plan

Plans:

- [x] 25-01: eBay credential docs added to .env.example with startup warning gating adapter visibility when creds are absent

**Details:**

**Goal**: Add 5 new Shopify stores plus Heartland Records fallback scraping path
**Depends on**: Phase 25

**Plans**: 2 plans

**Details:**

### Phase 27: Clarity Records Adapter

**Goal**: Add Clarity Records as a new scraping source using BigCommerce HTML scraping
**Depends on**: Phase 26
**Requirements**: SRC-13
**Plans**: 1 plan

Plans:

- [x] 27-01: BigCommerce HTML scraper for clarityrecords.com.au returning AUD-priced listings with stock detection via juno.py pattern

**Details:**
- `app/services/clarity.py` (131 lines): GETs BigCommerce `/search.php?search_query={query}`, parses with BeautifulSoup + lxml, handles 403/404 gracefully, rate-limits with Semaphore(1) + 2s sleep
- `app/services/adapter.py`: imports clarity and adds it as the 7th enabled entry in ADAPTER_REGISTRY
- Selectors: `li.product`, `.card-title a`, `.price--withoutTax`, `.card-out-of-stock` (Stencil defaults)
- Both in-stock and sold-out listings returned (`is_in_stock=False` for sold-out)

## Milestone Summary

**Key Decisions:**

- Phase 25 ordered first: eBay is a credential/config task, independent of scraping work; verifying it unlocks confidence in the full source list before adding more
- Phase 26 groups all Shopify additions (SRC-07–SRC-12) in one phase — same file, same surface area
- Phase 27 isolated as a new adapter with meaningfully different complexity (HTML pagination, new file, registry registration)
- Heartland: products.json fallback chosen over suggest.json (disabled on their Shopify store); client-side substring filter acceptable for targeted queries
- Clarity: page 1 only — for targeted artist/album queries, BigCommerce relevance ordering makes this sufficient (D-02)
- No pagination added to either new path — simplicity first, extend if coverage proves insufficient

**Issues Resolved:**

- Rockaway Records base URL: `.com.au` redirects; used `rockaway.com.au` directly
- Happy Valley base URL: `.com.au` connection-fails; store is hosted under `.com`
- Clarity Records domain: previously NXDOMAIN (hence adapter was disabled in v1.0); now live and accessible

**Technical Debt Incurred:**

- Heartland products.json fetches all 250 products per query and filters client-side — inefficient for generic queries but acceptable for targeted artist/album searches
- eBay adapter may return cassette/non-vinyl listings alongside vinyl — a future filter phase is backlogged
